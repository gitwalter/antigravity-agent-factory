## Express Lane

**Duration:** 10-15 minutes
**Team Size:** Individual
**Best For:** Developers who know what they want and want to get started quickly

### What You'll Do

1. **Welcome & Intent** (2 min) - Share your dream for this project
2. **Values Check** (2 min) - Pick 3 values that matter most
3. **Stack Discovery** (3 min) - Choose your technology with smart suggestions
4. **Agent Design** (3 min) - Meet your AI team
5. **Wisdom Capture** (2 min) - Share one lesson learned
6. **Magic Preview** (2 min) - See what you'll get before confirming
7. **Celebration & Launch** (1 min) - Generate with fanfare!

### How to Start

In Cursor, open a chat and say:

```
I want to create a new agent system using Express Lane onboarding
```

Or use the skill directly:

```
/skill express-onboarding
```

### What You'll Get

- Personalized `.cursorrules` with your values embedded
- `PURPOSE.md` capturing your mission
- Custom agents and skills for your stack
- `knowledge/team-wisdom.json` with your captured insights
- **Integrity Guardian** - Layer 0 protection for all AI operations
- **Memory System** - User-validated learning from your interactions
- **4 Layers of Value Propagation** - Foundation (verification), Development Autonomy (self-healing CI/CD), Knowledge Independence (extend knowledge), and optional PM Integration

## Existing Repository

**Duration:** 15-30 minutes
**Team Size:** Any
**Best For:** Adding AI agents to projects that already have code

### What Happens

1. **Analysis** - Factory detects your tech stack and existing artifacts
2. **Blueprint Matching** - Suggests the best blueprint for your stack
3. **Conflict Resolution** - Handles any overlapping configurations
4. **Enhancement** - Adds missing agents, skills, and knowledge
5. **Backup** - Creates rollback point for safety

### How to Start

```powershell
# Analyze your repo first
python cli/factory_cli.py --analyze C:\Projects\my-existing-repo

# Preview changes
python cli/factory_cli.py --onboard C:\Projects\my-existing-repo --dry-run

# Execute onboarding
python cli/factory_cli.py --onboard C:\Projects\my-existing-repo
```

Or in Cursor chat:

```
Please onboard my existing repository at C:\Projects\my-existing-repo
```

See **ONBOARDING_GUIDE.md** for full details.

## Understanding Value Propagation

Every generated project receives **4 layers of value** that provide ongoing capabilities beyond the initial generation. These layers ensure your project continues to benefit from Factory innovations even after onboarding.

| Layer | Name | What It Does | Commands |
|-------|------|--------------|----------|
| Layer 0 | Foundation | Verifies AI outputs, prevents hallucinations | "Verify this claim" |
| Layer 1 | Development Autonomy | Self-healing CI/CD, automatic debugging | "Monitor CI pipeline", "Fix pipeline errors" |
| Layer 2 | Knowledge Independence | Extend knowledge without Factory updates | "Extend knowledge for [topic]" |
| Layer 3 | PM Integration | Project management (opt-in) | "Create a story", "Plan sprint" |

### Layer 0: Foundation

**Automatic** - Included in every project

The Foundation layer provides integrity verification and hallucination prevention. The Integrity Guardian monitors all AI operations for axiom alignment and factual accuracy. This layer ensures that AI-generated code and documentation meet quality standards and align with your project's values.

**Try it:** Say `"Verify this claim"` when reviewing AI-generated code or documentation.

### Layer 1: Development Autonomy

**Automatic** - Included in every project

Development Autonomy enables self-healing CI/CD pipelines and automatic error detection. Your project can monitor its own CI/CD status, detect failures, and automatically fix common pipeline issues without manual intervention.

**Try it:**
- `"Monitor CI pipeline"` - Set up continuous monitoring
- `"Fix pipeline errors"` - Automatically diagnose and fix CI/CD failures

### Layer 2: Knowledge Independence

**Automatic** - Included in every project

Knowledge Independence allows your project to extend its own knowledge base without requiring Factory updates. You can add domain-specific knowledge, patterns, and best practices that become part of your project's permanent knowledge system.

**Try it:** `"Extend knowledge for [topic]"` - Add new domain knowledge that agents can reference

### Layer 3: PM Integration

**Opt-in** - Available when configured during onboarding

PM Integration connects your project to project management systems (Jira, Linear, GitHub Issues, etc.) and provides skills for creating stories, planning sprints, tracking burndown, and managing team workflows.

**Try it:**
- `"Create a story"` - Generate user stories with acceptance criteria
- `"Plan sprint"` - Pull stories into a sprint with capacity planning
- `"Run standup"` - Generate standup reports from current work

### Getting Started with Value Propagation

- **Layers 0-2 are automatic** - They're included in every generated project and work immediately
- **Layer 3 requires configuration** - Enable PM integration during onboarding or configure it later using the PM configuration skill
- **See **generated-output.md**** for complete details on all generated capabilities

## Upgrading Your Path

Started solo and now have a team? You can upgrade your setup.

### Express Lane to Team Huddle

If you completed Express Lane individually and now want team alignment:

1. Keep your generated project as-is
2. Run Team Huddle with your team
3. The huddle will enhance (not replace) your existing artifacts
4. Team values and wisdom merge with your individual setup

```
We completed Express Lane. Now we want to run Team Huddle to align as a team.
```

### Team Huddle to Workshop Series

For deeper alignment after a successful huddle:

1. Your huddle outputs become input for the workshops
2. Vision Quest can expand on your `TEAM_CHARTER.md`
3. Ethics Arena deepens the values you voted on
4. Run individual workshops as needed - you don't have to do all 5

```
We did Team Huddle last week. Now we want to run the Ethics Arena workshop.
```
